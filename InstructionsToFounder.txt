# Instructions to Founder - PocketAccountant Setup Requirements

Dear Founder,

To successfully develop and deploy PocketAccountant, we need you to set up the following accounts and credentials. These are essential for the development, testing, and eventual production deployment of the platform.

## 1. GitHub Account & Repository Setup

### Required Actions:
- [ ] **Create a GitHub account** (if you don't have one): https://github.com
- [ ] **Create a new repository** named "PocketAccountant"
- [ ] **Enable GitHub Actions** for CI/CD
- [ ] **Set up repository secrets** for:
  - AWS credentials (for deployment)
  - Docker Hub credentials (if using)
  - API keys (DeepSeek, Stitch, etc.)

### Repository Structure:
```
PocketAccountant/
├── backend/          # Node.js/Express API
├── frontend/         # React web application
├── mobile/          # React Native mobile app
├── workflows/       # n8n workflow exports
├── docker/          # Docker configurations
├── docs/           # Documentation
└── ProjectDocumentation/ # Existing documentation
```

## 2. DeepSeek API Account

### Required Actions:
- [ ] **Sign up for DeepSeek API**: https://platform.deepseek.com
- [ ] **Obtain API key** with sufficient quota for development
- [ ] **Note billing information** and set up payment method
- [ ] **Test API access** with sample prompts

### Usage Notes:
- We'll use DeepSeek for AI-powered categorization, Q&A, and tax assistance
- Consider the "pro" tier for higher rate limits during development
- Keep API key secure - we'll store it in environment variables

## 3. Stitch API Account (Bank Integration)

### Required Actions:
- [ ] **Sign up for Stitch API**: https://stitch.money
- [ ] **Apply for developer access** and sandbox environment
- [ ] **Obtain client ID and secret** for API authentication
- [ ] **Test with South African bank sandboxes**

### Important Notes:
- Stitch provides connectivity to major SA banks (FNB, ABSA, Standard Bank, etc.)
- We'll start with sandbox/testing environment
- Production access may require additional compliance checks

## 4. AWS Account Setup (Cape Town Region)

### Required Actions:
- [ ] **Create AWS account** (if you don't have one): https://aws.amazon.com
- [ ] **Enable Cape Town region** (af-south-1) for data sovereignty
- [ ] **Set up IAM user** with appropriate permissions for:
  - EC2/ECS (container hosting)
  - RDS (PostgreSQL database)
  - S3 (file storage)
  - CloudFront (CDN)
  - Route 53 (DNS)
- [ ] **Generate access keys** for programmatic access
- [ ] **Set up billing alerts** to monitor costs

### AWS Services Required:
1. **EC2/ECS**: For hosting n8n and backend services
2. **RDS**: PostgreSQL database (multi-AZ for high availability)
3. **S3**: Storage for receipts, documents, and backups
4. **CloudFront**: Content delivery for web app
5. **Route 53**: Domain management and DNS
6. **CloudWatch**: Monitoring and logging

## 5. Domain Name Registration

### Required Actions:
- [ ] **Register domain name** (suggestions: pocketaccountant.co.za, mypocketaccountant.co.za)
- [ ] **Set up DNS records** in Route 53
- [ ] **Configure SSL certificates** via AWS Certificate Manager

### Domain Suggestions:
- Primary: `pocketaccountant.co.za`
- Development: `dev.pocketaccountant.co.za`
- API: `api.pocketaccountant.co.za`

## 6. Docker Hub Account (Optional)

## Security Notes:

1. **Never commit credentials** to version control
2. Use **environment variables** and secret management
3. Enable **multi-factor authentication** on all accounts
4. Regularly **rotate API keys** and passwords
5. Maintain **access logs** for audit purposes
