# PocketAccountant Implementation Steps

## Phase 1: MVP (Months 1-2) - Core Bookkeeping + SARS Q&A

### Sprint 1 (Week 1-2): Foundation Setup
- [ ] Set up Docker environment for n8n, PostgreSQL, and backend services
- [ ] Configure DeepSeek API connection and test basic integration
- [ ] Implement user authentication system (email/password with JWT)
- [ ] Design and create core database schema (users, accounts, transactions, invoices)
- [ ] Build n8n workflow for manual transaction entry
- [ ] Create DeepSeek categorization workflow for transactions

**Test after Sprint 1:**
- Verify Docker containers are running (n8n, PostgreSQL)
- Test DeepSeek API connection with sample prompts
- Test user registration and login flow
- Verify database schema creation and basic CRUD operations

### Sprint 2 (Week 3-4): SARS Knowledge Base & AI Assistant
- [ ] Collect and organize SARS public documents (guides, ITR12 forms, TaxTim content)
- [ ] Set up vector database for RAG (Retrieval-Augmented Generation)
- [ ] Build document ingestion pipeline for SARS knowledge base
- [ ] Create n8n workflow for SARS Q&A: user question → retrieve context → call DeepSeek → return answer
- [ ] Implement conversation history and context management
- [ ] Create basic UI for chat interface

**Test after Sprint 2:**
- Test document ingestion with sample SARS PDFs
- Verify RAG retrieval returns relevant context
- Test SARS Q&A with common tax questions
- Validate conversation context is maintained

### Sprint 3 (Week 5-6): Invoice Generation & Dashboard
- [ ] Build invoice generator with PDF creation (n8n + PDF library)
- [ ] Create invoice management system (create, send, track)
- [ ] Develop basic React dashboard with P&L and recent transactions
- [ ] Implement transaction reconciliation interface
- [ ] Set up pilot user onboarding process
- [ ] Create user feedback collection mechanism

**Test after Sprint 3:**
- Generate sample invoices and verify PDF format
- Test invoice sending via email
- Verify dashboard displays correct financial data
- Test reconciliation workflow
- Onboard 3-5 pilot users successfully

**Test after Sprint 4:**

- Verify GPS data is received and stored correctly
- Test trip classification accuracy
- Validate mobile app builds for iOS and Android

### Sprint 5 (Week 9-10): Mileage Reporting & Manual Override

**Test after Sprint 5:**

- Generate SARS-compliant mileage reports

**Test after Sprint 6:**
