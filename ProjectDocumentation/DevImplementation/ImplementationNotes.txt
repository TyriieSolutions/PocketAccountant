# PocketAccountant Implementation Notes

## Project Start Date: February 25, 2026

## Technology Stack & Justification

### Core Technologies:
1. **n8n (Workflow Engine)** - Self-hosted
   - Low-code platform with 400+ integrations
   - Perfect for gluing services together without extensive custom code
   - Enables rapid prototyping and iteration

2. **DeepSeek API (AI/LLM)** 
   - Cost-effective compared to OpenAI
   - Strong reasoning capabilities
   - Can be fine-tuned on SARS documentation
   - Supports RAG (Retrieval-Augmented Generation)

3. **PostgreSQL (Database)**
   - Robust relational database
   - Handles double-entry accounting requirements
   - Strong ACID compliance for financial data
   - Can be hosted via Supabase or AWS RDS

4. **Node.js/Express (Backend API)**
   - For custom logic that n8n can't handle
   - Large ecosystem of libraries
   - Good performance for financial calculations

5. **React (Web Frontend)**
   - Component-based architecture
   - Large community and ecosystem
   - Good performance for dashboards

6. **React Native (Mobile)**
   - Code reuse between web and mobile
   - Access to native device features (GPS)
   - Single codebase for iOS and Android

7. **Docker (Containerization)**
   - Consistent development environment
   - Easy deployment to AWS
   - Isolation of services

### Integration Points:
- **Bank Feeds**: Stitch API (South African bank connectivity)
- **GPS Tracking**: react-native-background-geolocation plugin
- **Email/SMS**: n8n Email node + Twilio for notifications
- **PDF Generation**: Libraries like pdfkit or puppeteer
- **Vector Database**: Pinecone, Weaviate, or pgvector for RAG

## Key Design Decisions

### 1. AI-First Architecture
- All user interactions go through AI layer first
- AI handles categorization, recommendations, and Q&A
- Human override available for all AI decisions

### 2. South African Focus
- SARS tax rules as primary compliance framework
- ZAR currency as default
- Local bank integrations (Stitch for SA banks)
- POPIA compliance built-in

### 3. Mobile-First for Mileage Tracking
- Background GPS tracking optimized for battery life
- Offline capability for trip recording
- Sync when network available

### 4. Security & Compliance
- Data encryption at rest and in transit
- AWS Cape Town region for data sovereignty
- Audit trails for all financial transactions
- AI decision logging for explainability
